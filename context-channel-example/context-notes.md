- A set or statements those are executed across an application or API boundaries is called a context

- The context package is to facilitate the cancellation of operations when they are no longer needed or when they should be terminated for some reason.Example reached deadlines or timeouts.

- This is especially important in systems where you have concurrent or asynchronous operations running, and you want to ensure that resources are released and operations are stopped gracefully.

- There is a parent context and there are child contexts too.
